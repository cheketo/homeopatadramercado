Meta Tag
<meta name="author" content="www.ideas2.net">
<meta name="DC.Creator" content="www.ideas2.net">
Se utiliza para indicar el autor del sitio
<meta name=copyright content="ideas2.net">
<link rel="posicionamiento en buscadores " content="http://www.ideas2.net">
<meta name="Designer" content="ideas2.net">
Se utiliza para indicar el copyright del sitio
<meta name="distribution" content="all">
<meta name="robots" content="all">

El valor de content puede ser:

all para que el robot tenga en cuenta esta página y pueda seguir los links contenidos en la misma (este es el valor predeterminado)
noindex para que no la tenga en cuenta.
none que es lo mismo que noindex, nofollow
index para que la tenga en cuenta
follow para que pueda seguir los links que haya en la página
nofollow lo contrario de follow.
<meta name="revisit" content="15 days">
<meta name="REVISIT-AFTER" content="15 days">
Es para que el buscador nos visite nuevamente luego de tantos dias.
<title>Aire Acondicionado y Calefacción. Haverland. Calor AZUL</title>
<meta name="title" content="Aire Acondicionado y Calefacción. Haverland">
<meta name="DC.Title" content="Aire Acondicionado y Calefacción. Haverland">
<meta http-equiv="title" content="Aire Acondicionado y Calefacción. Haverland">
Es el titulo generico del sitio. Si deseamos poner un titulo especifico de una web, se usa la variable $meta_tag_title_opcional. Debe ser muy corto, ya que luego va cargado el nombre del producto o de la seccion.
No poner mas de 50 caracteres, Hay que poner los dos titulos.

<meta name="description" content="Suministro e instalacion de aire acondicionado. Equipos emisores termicos Haverland. Especialistas en calor azul">
<meta http-equiv="description" content="Suministro e instalacion de aire acondicionado. Equipos emisores termicos Haverland. Especialistas en calor azul">
<meta http-equiv="DC.Description" content="Suministro e instalacion de aire acondicionado. Equipos emisores termicos Haverland">

Muestra la descripcion generica del sito. Cuando esta definido la variable $meta_tag_description_opcional, reemplaza a $meta_tag_description, para evitar poner gran cantidad de caracteres.
No poner mas de 200 caracteres.
<meta http-equiv="keywords" content="Haverland, daikin, fujitsu, climatizacion domestica">
Son las palabres claves genericas, se usa, para cuando no se especifican. Tiene que estar separado por comas, ej: auto, lancha, compra, etc
No poner mas de 500 caracteres, o 20 palabras
<meta http-equiv="Content-Type" content="text/html; ISO-8859-1">
Es para definir el mapa de caracteres. El ISO-8859-1 reconoce los caracteres con hacento como ó ú etc..
